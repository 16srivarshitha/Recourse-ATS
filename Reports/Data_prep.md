# Recourse-ATS: Phase 1 Data Preparation & EDA Report
**Date:** February 24, 2026  
**Phase:** 1 (Data Engineering, Graph Construction, & Exploratory Data Analysis)

## 1. Project Overview
The objective of Recourse-ATS is to improve upon standard semantic embedding models (like Resume2Vec) by integrating a Graph Neural Network (GNN). This hybrid approach aims to provide both accurate Resume-Job Description (JD) matching and mechanistic interpretability (recourse) via a Skill Co-occurrence Graph. 

This report summarizes the data preparation, text cleaning, and baseline graph generation required before model training.

## 2. Dataset 1: Resume-JD Matching (Supervised Training Data)
**Source:** `cnamuangtoun/resume-job-description-fit` (HuggingFace)

*   **Total Samples:** 8,000 pairs (Train: 6,241 | Test: 1,759)
*   **Label Distribution:** 
    *   `No Fit` (0.0): 3,143 (50.3%)
    *   `Potential Fit` (0.5): 1,556 (24.9%)
    *   `Good Fit` (1.0): 1,542 (24.7%)
*   **Text Length Analysis (Pre-cleaning):**
    *   Resumes: Median = 618 words, 95th Percentile = 1,591 words.
    *   Job Descriptions: Median = 328 words, 95th Percentile = 810 words.
*   **Text Cleaning Impact:** By lowercasing, removing special characters, and filtering English stopwords, document lengths were reduced by **~24.9%**. This is critical for fitting combined inputs into the 512-token limit of RoBERTa/BERT architectures.

## 3. Dataset 2: Interpretability Validation (Backup)
**Source:** `AzharAli05/Resume-Screening-Dataset`
*   **Samples:** 10,174 rows.
*   **Key Finding:** This dataset includes a `Reason_for_decision` column (e.g., *"Lacked leadership skills"*). This will be utilized in Phase 2 to ground-truth and validate the explanations generated by SHAP/Integrated Gradients.

## 4. Dataset 3: Skill Graph Construction 
**Source:** `1.3M LinkedIn Jobs & Skills (2024)` (Kaggle)

To enable recourse, a Skill Co-occurrence Graph was built using real-world job postings to map the relationships between technical and soft skills.

*   **Vocabulary Generation:** 
    *   Processed 100,000 jobs, exploding comma-separated skill strings.
    *   Identified **376,422** unique, raw skill phrases.
    *   Created a core dictionary of the **Top 500 most frequent skills** (e.g., *communication, python, customer service, data analysis*).
*   **Graph Edge Generation:**
    *   Iterated through 50,000 job postings to find skills required simultaneously.
    *   Generated **241,244 unique co-occurrence edges**.
    *   *Sample Strong Edges:* Communication $\leftrightarrow$ Teamwork (Weight: 5,525), Problem Solving $\leftrightarrow$ Teamwork (Weight: 2,430).

## 5. Multi-Strategy Skill Extraction Validation
A fast dictionary-matching algorithm was tested on the cleaned Resume dataset using the Top 500 LinkedIn skill vocabulary. 

*   **Results:** Successfully extracted between 30 to 50 skills per resume.
*   **Verification:** The extraction correctly clustered domains. For example, Resume 2 (Engineering) successfully yielded domain-specific nodes: `['cad', 'electrical engineering', 'sap', 'specifications']`.

2.  **Baseline Modeling:** Implement a TF-IDF + Cosine Similarity baseline to establish the performance floor of traditional keyword-based ATS systems.
